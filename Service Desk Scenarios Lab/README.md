# Service Desk Scenarios Lab

## Purpose
The purpose of this lab is to demonstrate structured, real-world service desk thinking when handling common IT support issues. The lab focuses on documenting how issues are identified, investigated, resolved, communicated, and escalated in line with best practices for first-line IT support roles.

This lab emphasises problem-solving methodology and professional communication rather than technical setup.

---

## Tools Used
- Service desk best-practice workflows
- Active Directory concepts
- ITIL-aligned troubleshooting approach
- Documentation and communication skills

---

## Scenarios Covered
1. Locked Account
2. Slow Computer Performance
3. Cannot Access Shared Folder
4. VPN Not Connecting
5. Printer Not Printing

Each scenario follows a consistent service desk structure.

---

## Scenario Structure
Each scenario was documented using the following format:
- Issue Reported
- Initial Checks
- Troubleshooting Steps
- Resolution
- Escalation (if required)
- User Communication

This ensures consistency, clarity, and repeatability when handling support requests.

---

## Scenario Summaries

### Locked Account
- Verified user identity and account status.
- Checked account lockout in Active Directory.
- Unlocked the account and reset the password.
- Forced password change at next logon.
- Communicated resolution clearly to the user.
- Escalation considered for repeated lockouts.

---

### Slow Computer Performance
- Confirmed scope and impact of performance issue.
- Checked system resource usage and startup applications.
- Optimised startup items and restarted the system.
- Verified performance improvement with the user.
- Escalation identified for persistent hardware or software issues.

---

### Cannot Access Shared Folder
- Verified folder path and access error message.
- Checked user group membership and permissions.
- Reviewed share and NTFS permissions.
- Granted access through appropriate security group.
- Confirmed successful access with the user.
- Escalation identified for sensitive or complex permission structures.

---

### VPN Not Connecting
- Confirmed remote working status and internet connectivity.
- Checked VPN client configuration and credentials.
- Verified account status and group membership.
- Restored VPN access after configuration validation.
- Escalation identified for wider VPN service issues.

---

### Printer Not Printing
- Confirmed affected printer and user scope.
- Checked printer status, network connection, and print queue.
- Restarted printer and cleared stuck print jobs.
- Verified successful printing with the user.
- Escalation identified for suspected hardware faults.
