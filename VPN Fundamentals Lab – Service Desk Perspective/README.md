## Purpose
The purpose of this lab is to demonstrate a practical understanding of VPN fundamentals from a first-line IT service desk perspective. The lab focuses on client VPN configuration, common connection failures, structured troubleshooting, escalation decision-making, and professional user communication.

This lab reflects real-world VPN issues commonly handled by service desk analysts rather than network engineers.

---

## Tools Used
- Windows 10
- Windows built-in VPN client
- Basic service desk troubleshooting methodology
- Documentation and communication skills

---

## Lab Environment
- Device: Windows 10 virtual machine
- User context: Standard domain user
- VPN type: Client VPN (user-to-network)

---

## VPN Concepts Covered
- Client VPN fundamentals
- VPN authentication
- Common VPN connection failures
- First-line troubleshooting process
- Escalation criteria
- User communication best practices

## Tasks Performed

### VPN Profile Configuration
- Accessed Windows VPN settings.
- Created a client VPN profile using Windows built-in VPN functionality.
- Configured connection details including VPN provider, server address, and authentication method.
- Saved the VPN profile for user access.

## Why This Lab Matters
VPN issues are one of the most common problems handled by IT service desks, especially with remote and hybrid working. This lab demonstrates the ability to support VPN users professionally, apply structured troubleshooting, and escalate correctly when required.

The skills demonstrated are directly applicable to first-line and junior IT support roles.
