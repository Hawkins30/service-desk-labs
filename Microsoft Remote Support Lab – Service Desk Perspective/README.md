# Microsoft Remote Support Lab – Service Desk Perspective

## Purpose
The purpose of this lab is to demonstrate the ability to provide remote user support using Microsoft built-in tools from a first-line IT service desk perspective. The lab focuses on initiating a remote support session, obtaining user consent, performing basic troubleshooting actions, and ending the session securely.

This lab reflects real-world remote support tasks commonly handled by service desk and junior IT support roles.

---

## Tools Used
- Windows 10
- Microsoft Quick Assist
- Remote support best practices
- User communication and consent handling

---

## Lab Environment
- Support Technician Device: Windows system with Quick Assist
- User Device: Windows 10 virtual machine
- User Context: Standard user account
- Support Context: IT support technician

---

## Remote Support Concepts Covered
- Remote assistance using Microsoft Quick Assist
- Secure session initiation
- User consent and permissions
- First-line remote troubleshooting
- Professional session closure

---

## Tasks Performed

### Remote Support Session Setup
- Opened Microsoft Quick Assist on both the support technician and user devices.
- Generated a secure support code on the technician device.
- Entered the support code on the user device to request assistance.
- Obtained explicit user consent before initiating the session.
